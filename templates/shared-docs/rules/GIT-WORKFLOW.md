# Git Workflow

1. `git status` before any git action.
2. Never use `git add .` or `git add -A`.
3. Stage only explicit task-related files.
4. Ask user confirmation before commit and push.
5. Ask user how to handle unrelated local changes.
6. Always create a dedicated task branch before implementation.
7. Open a Pull Request and merge into `main` only through PR flow.
8. Never push directly to `main`.
