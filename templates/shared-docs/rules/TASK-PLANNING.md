# Task Planning

Before implementation clarify:
1. Deadline
2. Additional materials/constraints
3. Team capacity
4. Priority

Then create plan and request explicit approval.

## Discovery-First Requirement

Before any task plan:
1. Read project overview (`.ai/shared-docs/project-overview.md`).
2. Read existing project docs (`*.md` outside `.ai`).
3. Verify module presence by actual directories and source files.
4. Record evidence paths in plan notes.

Do not declare project "empty/template" from one file or one manifest.
If there is conflicting evidence, list conflicts and ask user to resolve before planning.
